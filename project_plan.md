# 基于频域径向功率谱与空间 LBP 纹理特征多级融合的 AIGC 图像轻量级检测系统

## 1. 项目目标

本项目面向《数字图像处理》课程期末实践，目标是在**不使用深层神经网络**的前提下，仅依赖传统图像处理与传统机器学习方法，实现一个能够区分 **真实图像（REAL）** 与 **AI 生成图像（FAKE）** 的二分类检测系统。

本项目核心思想如下：

1. 在**频域**中，通过二维傅里叶变换提取图像的频谱信息。
2. 通过**径向平均功率谱**压缩二维频谱，捕捉 AIGC 图像在高频区域的异常能量分布与周期性伪影。
3. 在**空域**中，通过 LBP 纹理统计刻画扩散模型生成图像局部纹理过于平滑的问题。
4. 将频域特征与空域特征进行融合，再交给传统分类器（SVM）进行学习与判别。

最终输出包括：

- 可运行的 Python 项目代码
- 训练与测试流程
- 分类评价指标
- 混淆矩阵与可视化结果
- 可用于答辩展示的实验结论

## 2. 项目总体技术路线

完整流程可以概括为：

1. 读取 CIFAKE 数据集图片
2. 图像预处理
3. 提取频域径向功率谱特征
4. 提取空域 LBP 纹理特征
5. 融合两类特征
6. 可选 PCA 降维
7. 使用 SVM 训练分类器
8. 在测试集上评估性能
9. 输出结果图表与实验结论

可视化流程如下：

```text
图像输入
   ↓
灰度化
   ↓
├─ 频域分支：FFT → 功率谱 → 径向平均 → 频域特征
└─ 空域分支：LBP → 归一化直方图 → 纹理特征
   ↓
特征拼接
   ↓
可选 PCA 降维
   ↓
SVM 分类器训练
   ↓
测试集评估与可视化
```

### 阶段 3：实现频域特征提取模块（FFT）

目标：

- 从灰度图中提取稳定的一维径向平均功率谱特征

建议文件：

- `src/features_fft.py`

核心流程：

1. 对灰度图做二维傅里叶变换：
   - `F = np.fft.fft2(gray)`
2. 做中心化：
   - `F_shift = np.fft.fftshift(F)`
3. 计算功率谱：
   - `P = np.abs(F_shift) ** 2`
4. 以频谱中心为圆心，按半径进行统计
5. 得到一维径向平均功率谱向量

建议实现的函数：

```python
def compute_power_spectrum(gray):
    pass

def radial_profile(power_spectrum):
    pass

def extract_fft_features(gray):
    pass
```

实现细节建议：

1. `power_spectrum` 可以先取对数，增强可视化效果：
   - `log_spectrum = np.log1p(P)`
2. 径向平均时，需要为每个像素点计算其到中心点的欧氏距离，再按整数半径分桶统计均值
3. 对输出向量做归一化，减少不同图像亮度差异的影响
4. 由于图像尺寸是 `32x32`，径向特征长度不会很大，适合传统模型

你们在这一阶段要重点验证的内容：

- REAL 与 FAKE 图像的平均径向谱曲线是否存在差异
- 高频尾部是否有明显异常突起
- 特征维度是否一致

建议可视化：

1. 原图
2. 灰度图
3. 频谱图
4. 径向平均曲线

阶段产出：

- 单张图像的 FFT 特征提取函数
- 批量提取 FFT 特征的脚本

### 阶段 6：PCA 降维与特征标准化

目标：

- 为传统分类器提供更稳定的输入表示

建议文件：

- `src/train.py`

为什么需要这一步：

- FFT 与 LBP 特征量纲可能不同
- 部分特征可能冗余
- SVM 对输入尺度较敏感

建议流程：

1. 先标准化：
   - `StandardScaler`
2. 再做 PCA：
   - `PCA(n_components=...)`
3. 保留累计方差贡献率，例如：
   - 95%
   - 或固定维度，如 32、48、64

建议实验设置：

1. 不使用 PCA，直接分类
2. 使用 PCA 保留 95% 方差
3. 对比两者性能差异

注意事项：

- `scaler` 只能在训练集上拟合
- `pca` 只能在训练集上拟合
- 测试集只能调用 `transform`

阶段产出：

- 标准化器
- PCA 模型

## 8. 可能遇到的问题与解决方案

### 问题 1：特征提取很慢

解决建议：

- 先只用小子集调试
- 完成后再跑全量数据
- 将特征缓存到磁盘

### 问题 2：FFT 特征维度不一致

原因：

- 图像尺寸不一致

解决建议：

- 强制 resize 到固定大小
- 径向统计时统一采用同样的半径范围

### 问题 3：SVM 效果不稳定

解决建议：

- 做标准化
- 调整 `C` 和核函数
- 尝试是否启用 PCA

### 问题 4：LBP 结果差异不明显

解决建议：

- 调整 `P`、`R`
- 试试 `P=16, R=2`
- 比较 `uniform` 与默认模式，但最终课程项目建议优先保留 `uniform`

### 问题 5：测试分数过高或过低

需要排查：

- 是否数据泄漏
- 是否错误地在全数据上 fit 了 scaler / PCA
- 是否训练集和测试集路径混用
